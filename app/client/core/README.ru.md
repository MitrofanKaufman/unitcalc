# Основной слой приложения

Эта папка содержит основную функциональность приложения, общие утилиты и фундаментальные компоненты, которые используются во всем приложении.

## Описание файлов

### RouteManager.ts
**Назначение**: Централизованное управление маршрутами и утилиты навигации.

**Функциональность**:
- Определяет все маршруты приложения и их конфигурации
- Управляет разрешениями маршрутов и контролем доступа
- Предоставляет валидацию маршрутов и обработку ошибок
- Обрабатывает генерацию динамических маршрутов
- Управляет метаданными маршрутов и навигационными цепочками
- Реализует охранники маршрутов и промежуточное ПО

**Ключевые особенности**:
- Централизованные определения маршрутов
- Контроль доступа на основе разрешений
- Генерация динамических маршрутов
- Валидация и санитизация маршрутов
- Генерация навигационных цепочек
- Интеграция границ ошибок

### constants.ts
**Назначение**: Константы и конфигурационные значения приложения.

**Функциональность**:
- URL-адреса и конфигурации конечных точек API
- Настройки и значения по умолчанию приложения
- Константы тем и стилизации
- Флаги функций и переключатели
- Конфигурации, зависящие от окружения
- Правила и ограничения валидации

**Ключевые особенности**:
- Конфигурация, зависящая от окружения
- Константы с безопасностью типов
- Централизованное управление конфигурацией
- Управление флагами функций
- Константы валидации

### messages.js
**Назначение**: Определения сообщений и строки локализации.

**Функциональность**:
- Пользовательские сообщения и метки
- Сообщения об ошибках и успешном выполнении
- Сообщения загрузки и состояния
- Сообщения валидации форм
- Уведомления и оповещения
- Строки интернационализации

**Ключевые особенности**:
- Централизованное управление сообщениями
- Поддержка нескольких языков
- Категоризация сообщений
- Динамическая генерация сообщений

### session.js
**Назначение**: Управление сессиями и состояние аутентификации пользователя.

**Функциональность**:
- Обработка и валидация пользовательских сессий
- Управление состоянием аутентификации
- Сохранение и восстановление сессии
- Обработка таймаута сессии
- Хранение пользовательских настроек
- Управление токенами безопасности

**Ключевые особенности**:
- Безопасная обработка сессий
- Автоматическое обновление сессии
- Сохранение пользовательских настроек
- Валидация сессии
- Ротация токенов безопасности

## Подпапки

### hooks/
Содержит основные пользовательские React хуки, используемые во всем приложении.

### interfaces/
Содержит определения интерфейсов TypeScript для основных структур данных.

### lib/
Содержит основные библиотечные утилиты и вспомогательные функции.

### scripts/
Содержит основные скрипты приложения и инструменты автоматизации.

### services/
Содержит основные классы сервисов и бизнес-логику.

### types/
Содержит основные определения типов TypeScript.

### utils/
Содержит основные утилитарные функции и помощники.

## Архитектурные заметки

- Основная функциональность отделена от UI компонентов
- Константы учитывают окружение и имеют безопасность типов
- Управление сессиями следует лучшим практикам безопасности
- Управление маршрутами централизовано и учитывает разрешения
- Управление сообщениями поддерживает интернационализацию
- Вся основная функциональность тщательно протестирована

## Зависимости

- React для интеграции компонентов
- TypeScript для определений типов
- API хранилища сессий (localStorage, sessionStorage)
- Библиотеки безопасности для управления токенами
- Библиотеки интернационализации (при наличии)

## Руководящие принципы использования

- Импортируйте основную функциональность с использованием алиаса `@/core/`
- Используйте константы вместо жестко закодированных значений
- Последовательно следуйте паттернам управления сессиями
- Используйте менеджер маршрутов для навигации
- Используйте константы сообщений для пользовательского текста
- Расширяйте основную функциональность вместо дублирования
