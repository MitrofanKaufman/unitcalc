# API слой

Эта папка содержит функциональность, связанную с API, для обработки внешних коммуникаций и получения данных в приложении.

## Описание файлов

### api.ts
**Назначение**: Основная конфигурация API и настройка базового API клиента.

**Функциональность**:
- Настраивает базовый URL API и конечные точки
- Устанавливает заголовки по умолчанию и аутентификацию
- Предоставляет экземпляр базового API клиента
- Обрабатывает общие конфигурации API
- Управляет версиями API и маршрутизацией

**Ключевые особенности**:
- Централизованная конфигурация API
- Управление токенами аутентификации
- Перехватчики запросов/ответов
- Обработка ошибок промежуточного ПО
- Поддержка версий API

### httpClient.ts
**Назначение**: Обертка HTTP клиента для выполнения API запросов.

**Функциональность**:
- Обертывает fetch API с дополнительной функциональностью
- Предоставляет согласованную обработку запросов/ответов
- Реализует логику повторных попыток для неудачных запросов
- Автоматически обрабатывает заголовки аутентификации
- Предоставляет логирование запросов/ответов
- Управляет таймаутами запросов и отменами

**Ключевые особенности**:
- Автоматическая повторная попытка при сбое
- Логирование запросов/ответов
- Обработка таймаутов
- Управление заголовками аутентификации
- Возможности кеширования ответов

### productService.ts
**Назначение**: Сервисный слой для операций API, связанных с товарами.

**Функциональность**:
- Поиск и фильтрация товаров
- Получение деталей товара
- Аналитика товаров и расчеты рентабельности
- Управление кешем для данных о товарах
- Обработка ошибок для товарных операций

**Ключевые особенности**:
- Поиск товаров с расширенными фильтрами
- Реализация стратегии кеширования
- Механизмы восстановления после ошибок
- Трансформация и нормализация данных

### suggest.ts
**Назначение**: Функциональность автодополнения и предложений.

**Функциональность**:
- Предложения названий товаров
- Завершение поискового запроса
- Получение предложений в реальном времени
- Ранжирование и фильтрация предложений
- Управление кешем для предложений

**Ключевые особенности**:
- Предложения поиска в реальном времени
- Дебаунсинг вызовов API
- Кеширование предложений
- Поддержка нечеткого поиска

### messages.js
**Назначение**: Обработка сообщений и интернационализация.

**Функциональность**:
- Управление пользовательскими сообщениями
- Обработка сообщений об ошибках
- Уведомления об успешных сообщениях
- Управление сообщениями загрузки
- Поддержка интернационализации

**Ключевые особенности**:
- Централизованное управление сообщениями
- Поддержка нескольких языков
- Форматирование сообщений
- Интеграция уведомлений тостов

## Подпапки

### db/
Содержит утилиты и конфигурации API, связанные с базой данных.

### n8n/
Содержит интеграцию с рабочими процессами N8N и API автоматизации.

### v1/
Содержит конечные точки API версии 1 и поддержку устаревшего API.

## Архитектурные заметки

- Все функции API следуют согласованным паттернам
- Обработка ошибок централизована и всеобъемлюща
- Стратегии кеширования реализованы во всех сервисах
- Аутентификация обрабатывается единообразно
- Трансформации запросов/ответов стандартизированы

## Зависимости

- Fetch API (стандарт браузера)
- Обертки пользовательского HTTP клиента
- Библиотеки кеширования (при наличии)
- Управление токенами аутентификации
- Утилиты обработки ошибок

## Руководящие принципы использования

- Используйте сервисные функции вместо прямых вызовов API
- Реализуйте правильную обработку ошибок в потребляющих компонентах
- Используйте кеширование для повышения производительности
- Последовательно следуйте паттернам аутентификации
- Тестируйте API функции независимо
