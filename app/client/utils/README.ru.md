# Утилитарные функции

Эта папка содержит утилитарные функции и вспомогательные методы, используемые во всем приложении для общих операций и преобразования данных.

## Описание файлов

### logger.ts
**Назначение**: Централизованная утилита логирования для отладки приложения и мониторинга.

**Функциональность**:
- Логирование консоли с различными уровнями (отладка, информация, предупреждение, ошибка)
- Структурированное логирование с метаданными
- Управление фильтрацией и уровнями логов
- Обработка логов в production/development
- Отслеживание ошибок и отчетность
- Логи мониторинга производительности

**Ключевые особенности**:
- Настраиваемые уровни логирования
- Структурированное логирование с контекстом
- Безопасное логирование для production
- Интеграция с границами ошибок
- Мониторинг влияния на производительность
- Фильтрация и поиск логов

**Уровни логирования**:
- `DEBUG`: Подробная отладочная информация
- `INFO`: Общие информационные сообщения
- `WARN`: Предупреждающие сообщения о потенциальных проблемах
- `ERROR`: Сообщения об ошибках для исключений и сбоев

**Использование**:
```typescript
import { logger } from '@/utils/logger';

// Различные уровни логирования
logger.debug('Отладочное сообщение', { context: 'компонент' });
logger.info('Информационное сообщение', { userId: 123 });
logger.warn('Предупреждающее сообщение', { issue: 'deprecated' });
logger.error('Сообщение об ошибке', { error, stack: true });
```

## Архитектурные заметки

- Утилиты являются чистыми функциями без побочных эффектов
- Логгер настраивается в зависимости от окружения
- Функции тщательно протестированы и задокументированы
- Утилиты следуют согласованным соглашениям об именовании
- Обработка ошибок встроена в утилитарные функции
- Учитываются соображения производительности

## Зависимости

- API консоли браузера для логирования
- Утилиты обнаружения окружения
- Библиотеки обработки ошибок (при наличии)

## Руководящие принципы использования

- Импортируйте утилиты с использованием алиаса `@/utils/`
- Используйте логгер последовательно во всем приложении
- Предпочитайте утилитарные функции встроенному коду
- Документируйте сложные утилитарные функции
- Тестируйте утилитарные функции независимо
- Учитывайте последствия производительности использования утилит
