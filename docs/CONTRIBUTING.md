# Руководство по вкладу в проект

## Структура проекта

```
unit.calculator/
├── app/                  # Клиентская часть приложения
│   ├── assets/           # Статические ресурсы (изображения, шрифты)
│   ├── components/       # Переиспользуемые React-компоненты
│   ├── pages/            # Страницы приложения
│   ├── hooks/            # Кастомные React-хуки
│   ├── lib/              # Вспомогательные библиотеки
│   ├── styles/           # Глобальные стили и темы
│   └── utils/            # Вспомогательные утилиты
│
├── server/               # Серверная часть приложения
│   ├── api/              # API-эндпоинты
│   ├── controllers/      # Контроллеры для обработки запросов
│   ├── middleware/       # Промежуточное ПО Express
│   ├── models/           # Модели данных
│   ├── routes/           # Определения маршрутов
│   └── services/         # Бизнес-логика
│
├── shared/               # Общий код между клиентом и сервером
│   ├── constants/        # Константы приложения
│   ├── types/            # Общие типы TypeScript
│   └── utils/            # Общие утилиты
│
├── config/               # Конфигурационные файлы
│   ├── app/             # Конфигурация приложения
│   ├── db/              # Конфигурация базы данных
│   ├── env/             # Переменные окружения
│   └── vite/            # Конфигурация Vite
│
├── docs/                # Документация
│   ├── api/             # API документация
│   ├── guides/          # Руководства
│   └── diagrams/        # Диаграммы архитектуры
│
├── logs/                # Логи приложения
│   ├── app/             # Логи приложения
│   ├── db/              # Логи базы данных
│   └── errors/          # Логи ошибок
│
├── public/              # Статические файлы
└── tests/               # Тесты
    ├── unit/            # Модульные тесты
    ├── integration/     # Интеграционные тесты
    └── e2e/             # End-to-end тесты
```

## Алиасы импортов

В проекте используются следующие алиасы:

- `@/*` - корневая директория приложения (`app/`)
- `@shared/*` - общие ресурсы (`shared/`)
- `@server/*` - серверная часть (`server/`)
- `@config/*` - конфигурация (`config/`)

Примеры использования:

```typescript
// Импорт из компонентов
import { Button } from '@/components/ui/Button';

// Импорт общих утилит
import { formatPrice } from '@shared/utils/format';

// Импорт сервисов
import { ProductService } from '@server/services/ProductService';
```

## Настройка окружения

1. Установите зависимости:
   ```bash
   npm install
   ```

2. Создайте файл окружения:
   ```bash
   cp .env.example .env
   ```

3. Запустите приложение в режиме разработки:
   ```bash
   npm run dev
   ```

## Стиль кода

- Используйте TypeScript для всего нового кода
- Следуйте принципам Clean Code
- Комментируйте сложную логику
- Используйте осмысленные имена переменных и функций

## Коммиты

Используйте Conventional Commits:

- `feat:` для новой функциональности
- `fix:` для исправления ошибок
- `docs:` для изменений в документации
- `style:` для форматирования, отсутствующих точек с запятой и т.д.
- `refactor:` для рефакторинга кода
- `test:` для добавления тестов
- `chore:` для обновления задач сборки, настройки пакетного менеджера и т.д.

## Процесс разработки

1. Создайте новую ветку от `main`
2. Внесите изменения
3. Напишите тесты для новой функциональности
4. Запустите линтеры и тесты
5. Создайте Pull Request

## Тестирование

Запуск всех тестов:
```bash
npm test
```

Запуск линтеров:
```bash
npm run lint
```

## Развертывание

Для продакшн-сборки:
```bash
npm run build
```

Для запуска продакшн-сборки:
```bash
npm run preview
```
