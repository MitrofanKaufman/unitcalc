# Структура страниц приложения

Этот документ описывает структуру и назначение страниц в приложении калькулятора рентабельности Wildberries.

## Общая структура

```
src/pages/
├── auth/                     # Страницы аутентификации
│   ├── Login.tsx            # Страница входа
│   └── Register.tsx         # Страница регистрации
├── calculator/              # Калькулятор рентабельности
│   ├── components/          # Компоненты, используемые только в калькуляторе
│   ├── CalculatorForm.tsx   # Форма калькулятора
│   ├── CalculatorResult.tsx # Результаты расчета
│   └── index.tsx            # Основной компонент калькулятора
├── dashboard/               # Личный кабинет
│   ├── components/          # Компоненты дашборда
│   ├── Analytics.tsx        # Аналитика продаж
│   └── index.tsx            # Основной дашборд
├── home/                    # Домашняя страница
│   ├── components/          # Компоненты домашней страницы
│   ├── HeroSection.tsx      # Герой-секция
│   ├── FeaturesSection.tsx  # Секция возможностей
│   ├── CTASection.tsx       # Призыв к действию
│   └── index.tsx            # Основной компонент домашней страницы
├── product/                 # Страницы товаров
│   ├── ProductDetails.tsx   # Детали товара
│   └── ProductAnalyze.tsx   # Анализ товара
└── shared/                  # Общие компоненты страниц
    ├── Layout.tsx           # Общий макет
    ├── NotFound.tsx         # Страница 404
    └── ErrorBoundary.tsx    # Обработчик ошибок
```

## Подробное описание страниц

### Аутентификация

- **Login.tsx** - Страница входа в систему с формой аутентификации
- **Register.tsx** - Страница регистрации нового пользователя

### Калькулятор рентабельности

- **CalculatorForm.tsx** - Форма для ввода данных о товаре и расчета рентабельности
- **CalculatorResult.tsx** - Отображение результатов расчета с детализацией издержек
- **components/** - Специфичные для калькулятора компоненты (формы ввода, карточки результатов)

### Личный кабинет

- **Analytics.tsx** - Аналитика продаж и рентабельности товаров
- **components/** - Виджеты и компоненты дашборда

### Домашняя страница

- **HeroSection.tsx** - Верхняя секция с основным призывом к действию
- **FeaturesSection.tsx** - Описание возможностей приложения
- **CTASection.tsx** - Секция с призывом к регистрации/входу

### Товары

- **ProductDetails.tsx** - Детальная информация о товаре
- **ProductAnalyze.tsx** - Анализ товара и расчет рентабельности

### Общие компоненты

- **Layout.tsx** - Общий макет приложения (хедер, футер, навигация)
- **NotFound.tsx** - Страница 404
- **ErrorBoundary.tsx** - Компонент для отлова и обработки ошибок

## Правила именования

1. Каждый файл должен содержать JSDoc-комментарий с описанием компонента
2. Имена файлов в PascalCase для компонентов React
3. Для страниц используется суффикс Page (например, `LoginPage.tsx`)
4. Компоненты, используемые только в рамках одной страницы, помещаются в подпапку `components/`

## Взаимодействие между страницами

Все переходы между страницами должны осуществляться через React Router. Импорты должны использовать алиасы, определенные в конфигурации Vite.
