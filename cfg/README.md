# Конфигурационные файлы

В этой директории хранятся все конфигурационные файлы проекта. 

## Структура

- `components.json` - конфигурация компонентов UI
- `esbuild.config.mjs` - конфигурация сборщика ESBuild
- `eslint.config.js` - конфигурация линтера ESLint
- `ormconfig.json` - конфигурация ORM (TypeORM)
- `playwright.config.ts` - конфигурация e2e тестов на Playwright
- `postcss.config.js` - конфигурация PostCSS
- `qodana.yaml` - конфигурация статического анализа кода Qodana
- `tailwind.config.ts` - конфигурация Tailwind CSS
- `tsconfig.clean.json` - дополнительная конфигурация TypeScript для очистки
- `tsconfig.node.json` - конфигурация TypeScript для Node.js

## Файлы, оставленные в корне

Некоторые конфигурационные файлы должны оставаться в корне проекта по разным причинам:

- `.env`, `.env.*` - переменные окружения (чувствительные данные, загружаются автоматически)
- `package.json` - основной файл конфигурации npm/yarn
- `tsconfig.json` - корневая конфигурация TypeScript
- `tsconfig.app.json` - конфигурация TypeScript для клиентской части
- `tsconfig.server.json` - конфигурация TypeScript для серверной части
- `vite.config.ts` - конфигурация сборщика Vite

## Обновление путей

При перемещении конфигурационных файлов необходимо обновить пути к ним в:

1. `vite.config.ts` - используйте функцию `configPath()` для указания путей
2. `package.json` - обновите пути в скриптах, если они ссылаются на перемещённые конфиги

## Примечания

- Все пути в конфигурационных файлах должны быть относительными к корню проекта
- При добавлении новых конфигурационных файлов добавляйте их в эту директорию и обновляйте документацию
- Чувствительные данные (пароли, ключи API) должны храниться в `.env` файлах и не коммититься в репозиторий
